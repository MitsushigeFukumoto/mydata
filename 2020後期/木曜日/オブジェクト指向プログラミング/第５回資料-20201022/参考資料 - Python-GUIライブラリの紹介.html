Python GUIライブラリの紹介

PythonでGUIアプリを作る場合の選択肢

PythonでGUIアプリ(デスクトップ向けアプリケーション)を作るには、大抵の場合ライブラリを利用することになります。

この資料は、GUIライブラリの代表的なものをいくつか紹介して、必要に応じて選択ができるようにする目的があります。 作成しようと思っているアプリケーションによって、使用するライブラリを選定していただければ幸いです。

Pythonでの代表的なGUIライブラリ

この講義では、GUIアプリの作成にTkinterを使用しました。 TkinterはPython公式のライブラリに標準搭載されており、ライブラリ自体を別途インストールする必要のないGUIライブラリです。 講義ではMacOSを使用する関係上、MacOS上のTcl/Tkライブラリのインストールや細かな設定変更が必要でしたが、これでも標準ライブラリの一つです。

Tkinterも含めてPythonで利用できる代表的なGUIライブラリを紹介します。

  1. Tkinter
    • Python標準ライブラリ
    • 古くからあるTclというスクリプト言語を使ってGUIライブラリを作成するTkと呼ばれるツールを使用するためのPythonライブラリで、PythonのコードをTcl/Tkに変換することで動作します。MacOSでそのまま動作しなかったのは、Tcl/Tk自体がMacOSのライブラリ群から外されたり設定が変わっていたりで面倒なことになっていました。
    • 参考リンク : tkinter --- Tcl/Tk の Python インタフェース — Python 3.9.0 ドキュメント
  2. Kivy
    • クロスプラットフォーム対応のGUIライブラリ
      • Windows,MacOS,Linuxだけでなく、AndroidやiOS向けのアプリ開発もできるらしいが、私は詳しくありません。
    • グラフィックの描写にOpenGLを用いることで、動作の軽量化やゲーム開発などにも対応できる
    • デメリットは、GUIの標準パーツに少し癖がある。しかし、それを許容できれば問題はない
    • 参考リンク : Kivy: Cross-platform Python Framework for NUI Development
  3. PyQt
    • Qtと呼ばれるGUIライブラリをPythonで使えるようにしたライブラリ
      • Qt自体はC++で記述されているため敷居が高く使用しづらいものだが、こちらはPythonコードで書けるため重宝されている。
    • GUIの各パーツやコンポーネントが洗練されており、ネイティブアプリと遜色ないデザインのアプリが作れる
    • 現時点では、PyQt4とPyQt5でバージョンが混在しているが、Python 3.8.5を使用するならPyQt5で良いと思われる
    • デメリットはPyQtはGPLライセンスを採用している関係で、商用利用は有償となる点で、どうしても無料で商用利用したい場合は、機能が少し制限されているPySideというライブラリがあるためそちらを使用すること(PySideの機能制限は意図的なものか単に開発が遅いだけかは判断ができていません)
      • ライセンスについてはこちらを参照してください : GNUライセンスに関してよく聞かれる質問 - GNUプロジェクト - フリーソフトウェアファウンデーション
    • 参考リンク : PyQt - Python Wiki
      • インストールから動作確認のチュートリアルにvenvを使用する旨が記載されているが、venvはPythonライブラリの使用バージョンを細かく分けるためのプロジェクト構築系ライブラリです。これを使う前には、venvについて少し調べてから使用してください。
  4. wxPython
    • wxWidgetと呼ばれるGUIライブラリのPython版
      • wxWidgetもC++
    • 他のGUIライブラリと比べるとアプリの動作が安定しているらしい
    • Tkinterと比べて、追加されたコンポーネントが豊富で、外部ツールに優秀な画面デザイン作成ツールがあるのが魅力(他のGUIライブラリにもデザイン作成ツールはあるが、使用感がそこまで良くない)
      • wxFormBuilder/wxFormBuilder
    • 参考リンク : Welcome to wxPython! | wxPython

代表的でないもの関しては割愛いたします。

GUIライブラリの選定方法

各GUIライブラリを使用したサンプルプログラムやGitHubなどで公開されている実際に作成されたアプリケーションのコード等をよく見て、実行結果のデザイン、コードの書きやすさ、拡張のしやすさなど、開発しようとしているアプリケーションにうまく適合するライブラリを選んでください。

ライブラリの選定自体がプロジェクト開発の足かせになる場合があるので、選定は慎重に行いましょう。

Webアプリケーションとデスクトップ向けGUIアプリケーション

Webアプリケーションは、Webブラウザ上で動作するアプリケーションのため、正しくクロスプラットフォームなアプリケーションと言えます。 昨今のアプリケーション開発事情で言うと、わざわざGUIライブラリを使用して開発するよりは、Webでいいよね？となるケースが多いように思います。

ただし、もちろんのことながらWebアプリケーション、GUIアプリそれぞれメリット・デメリットが存在しているのでそれらをよく理解した上で、Webかデスクトップ向けGUIアプリかを選びましょう。

例えば……

Webアプリケーションのメリット:

  • 画面の構築はHTML,CSS, JavaScriptなどのWeb標準技術である
  • Webブラウザで動作し、Webブラウザは大抵のプラットフォームで標準搭載されているため導入が容易
  • デスクトップアプリケーションや、モバイル向けアプリケーションもWeb技術の延長で作れなくもない

Webアプリケーションのデメリット:

  • Webアプリケーションをインターネット上で動作させるには、必ずWebアプリケーションを動作させるためのWebサーバーが必要で、基本的に有料である
  • 通信仕様のHTTPを勉強することが必須かつ、ある程度のコンピュータネットワークに対する知識が必要
  • HTML,CSS,JavaScriptを覚えなくてはならない
  • 新機能や新仕様の開発ペースが他と比べると格段に早く、常に新しいことに対して敏感にならないと置いていかれる可能性が高い

それに対してデスクトップアプリケーションは、配布さえできればどうにでもなります。

状況に応じて、デスクトップアプリケーションかWebかを選びましょう。

(※講師の永谷はWebを猛プッシュします。)
