事前準備

JavaScriptを書く場所

CSSと同じく、内部埋め込みか、外部ファイルとするかの2パターンがあります。

  • 内部埋め込みする場合
    • <script></script>タグを用いて、その内部にJavaScriptを書きます。
    • <head></head>内、<body></body>内どちらにも書くことができます。
  • 外部ファイルとする場合
    • 一般的にファイル名には、.jsという拡張子を付けます。
    • HTMLでは、scriptタグのsrc属性にファイルへのパスを指定します。
      • <head>〜</head>内に<script src="script.js" defer></script>のように書きます。
      • defer属性をscriptタグに追加しておくと、外部ファイルに限り対象ファイルのプログラムの実行タイミングをDOM読み込み後にずらすことが可能です。

HTMLに直接埋め込む

HTMLに直接書き込む場合は以下の様にscriptタグを用います。

<!doctype html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>JavaScript learning</title>
</head>
<body>
    <h1>JavaScript learning</h1>
    <script>

        //ここにJavaScriptのコードを書きます。

    </script>
</body>
</html>

実際に以下のコードを内に書き込んで、ブラウザで開いて動作確認してみましょう。

window.alert("Hello, World!!");

ブラウザで動作確認すると、ブラウザの警告ウィンドウで「Hello, World!!」と表示されます。

※HTML上にscriptタグでコードを埋め込む場合、scriptタグの位置はbodyタグの閉じタグ直前がおすすめです。

外部スクリプトを埋め込む

別途用意するJavaScriptファイル「script.js」をHTMLに埋め込む場合は、以下のようにscriptタグのsrc属性を使用して、JavaScriptファイルのあるファイルパスを指定します。

「script.js」のファイル名で、以下の1行だけ書き、HTMLファイルと同じ階層に保存しておいてください。

window.alert("Hello, JavaScript!!");

HTMLを以下のように変更します。

<!doctype html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>JavaScript learning</title>
    <script src="script.js" defer></script><!--外部ファイルを読み込み 実行タイミングはDOM読み込み後 -->
</head>
<body>
    <h1>JavaScript learning</h1>
</body>
</html>

ブラウザで動作確認すると、メッセージボックスで「Hello, JavaScript!!」と表示されます。

HTMLとJavaScriptを別のファイルに記載させるのには、それぞれを別のファイルとすることで管理しやすくさせる目的や、同じプログラムを複数箇所で書かないようにしてプログラミングのミス(バグ)を少なくする目的があります。

また、scriptタグにdefer属性を付与することでJavaScriptの実行タイミングをHTML読み込み完了後にずらすことができます。※src属性とセットで使用しないと機能しません。

JavaScriptの5つのポイント

  1. コード(命令文)の記述
    • Webブラウザに実行してもらいたい内容を記述します。
  2. コードを実行したい順番に上から並べて書く
    • JavaScriptのプログラムは基本的に上から下へ順にコードが実行されます。
  3. 1つのコードは、「;(セミコロン)」で終わる
    • 複数のコードを書く場合には、1つのコードの終端に必ず「;(セミコロン)」を入れましょう。
  4. コードのアルファベットの大文字小文字は区別される
    • コードのアルファベットは、大文字と小文字で別の文字として認識されます。
    • JavaScriptの文法上、大文字小文字が混在する場合もありますので間違いのないようにしましょう。
  5. 文字列は「'(シングルクォーテーション)」や「"(ダブルクォーテーション)」で囲う
    • 表示したいメッセージなどの文字列は、他のコードと区別するためにきちんと囲いましょう。

簡単なプログラムを動かしてみましょう

以下のHTMLを用意してください。

<!doctype html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>JavaScript learning</title>
</head>
<body>
    <h1>JavaScript learning</h1>
    <script>

        window.alert("こんにちは！");

    </script>
</body>
</html>

このプログラムでは、『「windowオブジェクト(ページをひらいている画面自体)」で「こんにちは！とアラート表示をしてください。」』という命令が書かれています。

このように、JavaScriptでは、〇〇で✕✕するという命令を、「〇〇.✕✕(✕✕する内容)」というように表します。

JavaScriptの記述に必要な基礎書式

前述の通り、プログラムの命令は、何をどうするという形式で書く必要があります。

  • 操作する対象は「オブジェクト」
    • 上の例では、「window」という画面自体を表すオブジェクトに対して処理を行っています。
    • 上記例以外では、HTMLの要素自体などもオブジェクトと言えます。
  • オブジェクトの動作を示す「メソッド」
    • 上の例では、「alert()」と言う警告表示を行う動作を行っています。
  • オブジェクトの状態を表す「プロパティ」
    • プロパティは、そのオブジェクトが持つ状態を表し、値の取得や、そのプロパティに対して何らかの操作を行うことでオブジェクトを変化させる働きを加えることができます。

JavaScriptデバッギング

JavaScriptで書いたプログラムの動作を確認する方法として、いくつかの方法を書いておきます。

アラートボックス表示

window.alert("メッセージ");

上記のように、JavaScriptのalertを用いる方法です。
処理の途中経過などを、alertで表示させながらプログラムの進行状況を確認するときなどに使用します。

アラートウィンドウが出ている間は、JavaScriptの処理自体は止まっています。

コンソール出力

Google Chromeで[option]+[command]+[j]を同時に押すと、JavaScriptコンソールが開きます。

ここにログデータとして出力するには、以下のようにJavaScriptを書きます。

console.log("メッセージ");

alertでは、呼び出されたタイミングでメッセージボックスが表示され、プログラムの実行がそこで一旦止まってしまいますが、このconsole.logなら処理を止めることなくJavaScriptコンソールのログとして処理の経過を表示させることができます。

Webページ上に出力

JavaScriptのHTML要素を操作する手法を用いてWebページ上にメッセージを出力させます。

HTMLファイルの内に準備が必要です。 <div id="message"></div>を追加してください。

以下のscriptタグをHTML内に書き込んでブラウザで開き、動作確認をしてみましょう。

    document.getElementById("message").textContent = "メッセージ";

今回は、HTMLで中身を入れなかったdivタグ内に「Hello, World!!」という文字列が入ります。 このように、JavaScriptからHTMLの要素の中身にアクセスし、値を操作することも可能です。

デバッギングは必要に応じて行いましょう。

自分で書いたプログラムの動作でおかしな動きをしている場合の原因究明には、デバッギングが必要になります。 Google Chromeなど、最新のブラウザなら、エラーログ表示機能などもあるのでそれを利用してもらっても良いですし、紹介した3つのメッセージ表示処理などを行い、処理の途中経過などを表示して、おかしな動きをしている原因を探すのもひとつの手です。

JavaScript付きのWebページをリリースする前には、JavaScriptのエラーやバグなどは極力なくすようによく確認をしましょう。

変数編

プログラミングで、非常に重要となる変数について勉強していきましょう。

変数とは

変数とはデータに付けるラベルのことです。

変数は宣言して使う

JavaScriptでは、変数を使う場合、「今からこの名前の変数を使います」という宣言をします。書き方は以下のようになります。

var text;

// ↓はECMAScript2015での新しい変数宣言
const hoge;
let fuga;  // よくわからない内は、letを使うと良いでしょう

古い書き方ですと、varという構文を使用します。スペースを開けて書かれている文字列が「変数名」となります。ここでは、「textという名前の変数を宣言しました。」ということになります。

データに変数を割り当てましょう

データに変数を割り当てる場合は、以下のように=(代入演算子)を使います。

let text;
text = "✌('ω'✌ )三✌('ω')✌三( ✌'ω')✌";
document.getElementById("message").textContent = text;

保存して、ブラウザで開くと、変数に割り当てたデータがきちんと表示されていることが確認できます。

変数の宣言と、代入は一行にまとめることもできます。

let text = "✌('ω'✌ )三✌('ω')✌三( ✌'ω')✌";
document.getElementById("message").textContent = text;

また、複数の変数を同時に宣言することも可能です。そう言う場合は、変数をカンマで区切り、以下のように書きます。

let text1, text2 = "text2", text3 = "text3";

JavaScriptのデータ型について知ろう

JavaScriptのデータには、幾つかの型があります。

主要なものを幾つか紹介します。

  • 文字列
  • 数値
  • 真偽 (true or false)
  • 未定義 (Undefined)
  • 無 (null)
  • 配列 (Array)
  • オブジェクトなど

それぞれどのように宣言すれば、その型となるか、以下にまとめます。

//文字列 ： ダブルクォーテーションもしくはシングルクォーテーションで括ると文字列となる。
let str = "文字列";

//数値　： 数字のみの場合は数値となる。
let num = 10;
let num2 = -5;          //マイナスも扱える。
let num3 = 0.2;         //小数も扱える。

//真偽 ： true もしくは false でフラグとして扱える。
let flg = true;
let flg2 = false;

//未定義 ： なにも定義されていない状態を表す。
let data1 = undefined;

//無 ： なにもない。
let data2 = null;

//配列 ： 1つの変数名に、複数のデータを割り当てたい場合に使用します。
let arr = new Array();
let arr2 = []; // これも配列定義
let array = new Array(3);
array[0] = "Sun";
array[1] = "Mon";
array[5] = "Fri";
array.push("Sat");  //末尾に追加
for(let i = 0; i < array.length; i++) {
  window.alert(array[i]);
}
// 以下のコードで初期化も可能
let array2 = ["Sun", "Mon", "Tue"];

// オブジェクト ： JavaScriptでは全てがオブジェクトなので、関数もオブジェクト
let obj = {};
obj.name = "hoge";
obj.value = 0.1;
obj.func = function(message){
  window.alert(message);
};
// ↓の書き方は、↑と同じ意味
let obj = {
  name: "hoge",
  value: 0.1,
  func: function(message){
    window.alert(message);
  },
}

JavaScriptでは、すべての変数や関数はオブジェクトです。この辺は多少込み入った話になるので、ここでは割愛します。

ES2015 新しい変数宣言

ECMAScript 2015より、以下の変数宣言構文が追加されました。

// 再代入不可変数の定義
const hoge = 123;
// hoge = 567; <- これ以降は再代入NG

// 再定義不可変数の定義
let fuga = "fuga";
// let fuga = "fugafuga"; <- これ以降は再定義NG
// var の場合↑は可能

何でもありのvarよりも、新しい仕様で追加されたconst, letのほうがルールが厳密で、かつ、他のプログラミング言語の仕様に近いため、極力const, letを使うようにしましょう。

演算子編

演算子は、データに対して何らかの計算を行うものです。

代入演算子 ( = )

実はもう既に使っています。=がそれです。

let num = 10;

文字列に関する演算子 ( + )

文字列の連結には+を使います。

let str = "Hello, " + "World!!";

数値に関する演算子 ( + - * / )

数値に対して四則演算を行う場合には、加算は+、減算は-、乗算は-、除算は/を使います。

let num1 = 1 + 2;   //num1 は 3
let num2 = 1 - 2;   //num2 は -1
let num3 = 1 * 2;   //num3 は 2
let num4 = 1 / 2;   //num4 は 0.5
//除算の余りを求めるには % を使います。
let num5 = 11 % 5;  //num5 は 1

また、特殊な書き方として、x = x + 1;を、x++と書いたり、y = y - 1;を、y--と書いたりします。

頻繁に出てくる書き方なので、覚えておきましょう。また、注意点として、x++、y--とすると、x,yそれぞれ値が変わってしまっています。気をつけましょう。

代入を伴う演算子 ( += -= *= /= )

また、特殊な書き方として、x = x + 5;など、同じ変数に対して何らかの演算を行う場合は、代入演算子と組み合わせて以下の様にも書けます。

let x = 4;
x += 5; //x = x + 5 と同じ xは9になる。

四則演算どれでも使うことができます。

この辺りは基本なので、きちんと身につけておきましょう。

条件分岐編

プログラミングでは、基本中の基本となる、条件分岐について学んでいきます。

条件分岐 ( if )

条件分岐は、if文にて行います。これは、「とある条件を満たしている場合に処理を実行します。」という処理の分岐ができます。コードは以下のとおりになります。

let message = "";   //画面表示用文字列 (空文字を代入して初期化してます)

let age = 18;       //年齢
if ( age < 20 ) {
    message = "未成年です。";
    if ( age >= 18 ) {
      message += "ですが選挙権はあります！";
    }
}

document.getElementById("message").textContent = message;

このコードでは、年齢を比較して、成人しているかどうかを判断しています。 ageの値を変えて、いろいろと試してみましょう。

if(……)の中にある比較が返す真偽(true or false)によって、trueの場合のみ{ }の中の処理に入ります。

比較演算子

if文に使える比較条件は、比較演算子によって決定します。以下に一覧を載せます。

比較演算子 説明 真(true)を返す例(var1 = 3, var2 = 5とします。)
==(等しい) 両辺のデータが等しい場合。(数値と文字列でも比較は可能) 3 == var1,"3" == var1,3 == '3'
!=(等しくない) 両辺のデータが等しくない場合。(数値と文字列でも比較は可能) var2 != 3,var2 != "3"
===(厳密に等しい) 両辺のデータが等しい場合。(データ型が違う場合は等しくないとされる) 3 === var1
!==(厳密に等しくない) 両辺のデータが等しくない場合。(データ型が違う場合は等しくないとされる) var2 !== 3,var2 !== "5"
>(より大きい) 左辺が右辺より大きいならば真 var2 > var1
>=(以上) 左辺が右辺以上ならば真 var2 >= var1,var2 >= 5
<(より小さい) 左辺が右辺より小さいならば真 var1 < var2
<=(以下) 左辺が右辺以上ならば真 var1 <= var2,var2 <= 5

条件分岐 ( if ~ else if ~ else )

if文にて偽(false)だった場合に対して、それに対して処理を行う場合は以下の様にします。

let message = "";   //画面表示用文字列 (空文字を代入して初期化してます)

let age = 20;       //年齢
if ( age < 20 ) {
    message = "未成年です。";
} else {
    message = "成人してます。選挙へ行こう。";
}

document.getElementById("message").textContent = message;

このコードでは、年齢を比較して、成人しているかどうかを判断しています。成人している場合は、選挙へ行く旨を伝えます。

また、if文にて偽(false)だった場合に対して、更に条件分岐を付ける場合もできます。

let message = "";   //画面表示用文字列 (空文字を代入して初期化してます)

let score = 60;     //点数
if ( score >= 60 ) {
    message = "合格！おめでとう！！";
} else if ( score >= 40 ) {
    message = "不合格です。残念。";
} else {
    message = "もう一年頑張りましょう。";
}

document.getElementById("message").textContent = message;

このコードでは、点数に対して合否を告げます。 また、else ifはいくつでもつなげることができます。

条件分岐 ( switch )

特定の１つのデータに対して、選択肢が複数ある場合はswitch文での条件分岐が便利です。

let message = "月末は ";   //画面表示用文字列

let month = 1;      //月
switch (month) {
    case 2:
        message += "28日（29日）";
        break;      //忘れずに書きましょう。
    case 4:
    case 6:
    case 9:
    case 11:
        message += "30日";
        break;
    default:        //上記以外の場合
        message += "31日";
        break;
}
document.getElementById("message").textContent = message;

switch()では、caseに指定した値と等しい場合は、break;がある行まで続きの処理を実行します。

caseは、上記のように続けて書くことで複数の指定もできます。比較演算子は使えません。また、どの条件にも当てはまらない場合は、default:の部分が実行されます。

ループ編

これも基本中の基本、ループ処理について学んでいきます。

whileループ

以下のコードを実行してみましょう。

let message = "計算内容: "; //画面表示用文字列
let answer = 0;             //結果出力用数値

let i = 0;
message += String(answer);

while (i < 20) {
    console.log(i);     //コンソールにループ対象となる値を出力
    i++;    // i = i + 1;
    message += " + " + String(i);
    answer += i;
}

message += " 計算結果: " + String(answer);
document.getElementById("message").textContent = message;

上記コードでは、0~20までを加算しつづけます。

whileでは指定した条件が真(true)である限り、ループを繰り返します。

Ctrl + Shift + jを押して、どの値でループを抜けているか確認をしてみましょう。

forループ

whileと同様に、ループを行うfor文を学びましょう。折角なので、while文と同じ条件で実行してみます。

let message = "計算内容: "; //画面表示用文字列
let answer = 0;             //結果出力用数値

message += String(answer);
for (let i = 0; i < 20; i++) {
    console.log(i);     //コンソールにループ対象となる値を出力
    message += " + " + String(i);
    answer += i;
}

message += " 計算結果: " + String(answer);
document.getElementById("message").textContent = message;

forでは括弧内に3つの処理を書くことができます。ここに書く内容は決まっていて、「ループ要素の変数(カウンタ)を初期化」、「ループを抜ける条件」、「ループカウンタの加算」の3つです。

実行結果が先ほどのwhileと異なっているのがわかるかと思います。

同様にGoogleChromeで[option]+[Command]+[j]を押して、どの値でループを抜けているか確認をしてみましょう。

両方のコードを特に条件の部分について、いろいろ書き換えてみて、実行結果がどのように変わるか確かめてみましょう。

条件分岐とループとそれぞれの命令

大抵のプログラムは細かいところまで拡大すれば、これらの組み合わせでできています。

どのプログラミング言語でも、最初に条件分岐とループの書き方を覚えることになると思いますので、きっちりと押さえておきましょう。

関数編

同じコードを複数箇所に書くのは、手間がかかりますし、バグも残しやすいです。
コードをまとめたり、計算処理を統合したりするには関数を使います。

実際にコードを見てもらったほうが良いと思うので、以下を御覧ください。

//aとbを加算し返します。
function sum(a, b) {
    return (a + b); //関数の呼び出し元へデータを返す（戻り値）
}

//div#consoleに指定したメッセージを表示させます。
function setMessage(str) {
    document.getElementById("message").textContent = str;
}

let answer = sum(48, 25);
setMessage(String(answer));

関数には引数と呼ばれるデータの受け渡し口が用意できます。
上のコードで言うとsum(a, b)のaとbです。これは、varで変数宣言したことと同義で、関数を呼び出す側にてデータを指定します。let answer = sum(48, 25);の中では、48, 25のところです。

ちなみに、引数はなしでも大丈夫です。

関数を右辺として代入演算子を使用したい場合は、return構文にて必ず戻り値を指定する必要があります。

また、関数の宣言方法として、以下のような書き方も可能です。

// 関数自体を変数として定義可能
const sum = function(a, b) {
    return (a + b);
};
// 実行方法は、sum(48, 25); などのように関数名のあとに括弧をつける

// ES2015より、以下の書き方も可能となっています。
const sum = (a, b) => {
    return (a + b);
};
// function() { ... } の書き方を、 () => { ... } として省略表記ができる

JavaScriptにおける関数は、定義のための構文や、変数への代入が可能など、特徴的な書き方ができます。 慣れないうちは分かりづらいところもありますので、function hoge() { ... }の書き方に統一しておくことが無難かと思います。

const func = function() { ... };という書き方は、他のプログラミング言語でも似たような書き方で実現できるものもあります。 その場合は、匿名関数という名前がついていたりします。

グローバル変数とローカル変数

変数には、それが使える範囲というものが決まっています。以下のコードを参照してください。

let g = "グローバルだぜ";

function f(a) {
    let l = "ローカルだぜ";
    setMessage(g + ":" + l + " " + a);
}

f("引数もローカルだぜ");
setMessage(l); //これはエラーになる
setMessage(a); //これもエラーになる

//div#consoleに指定したメッセージを表示させます。
function setMessage(str) {
    document.getElementById("message").textContent = str;
}

関数のコードブロックfunction xxx(～){～}内で宣言された変数及び引数は、その外で参照することはできません。また、別の関数では同じ名前の変数名にしてもエラーとなりません。（グローバルで宣言された変数は関数内で再宣言するとエラーになります。）

こういった変数の扱える(見える)範囲をスコープと呼びます。覚えておきましょう。
