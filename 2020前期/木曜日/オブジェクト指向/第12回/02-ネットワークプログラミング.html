ネットワークプログラミングの前に

コンピュータ・ネットワークに対する知識をつけましょう。

Webアプリケーションもネットワークプログラミングの一種

他の講義で行うようなWebアプリケーションも、ネットワークプログラミングに含まれます。 Webアプリケーションは、ネットワークの通信にHTTPというプロトコル（通信規約）を使用しており、Webクライアント（Webブラウザ）にてURIを指定することがプログラムを実行する手順となります。 言ってしまえば、Webアプリケーションがあれば、大抵のネットワーク経由で実行するプログラムはできてしまいますが、WebアプリケーションのようにHTTPによる高水準アクセスにて実行されるプログラムでは、できることに制限が付きます。

例えば、独自のプロトコルを持つメールサーバーでのメールの送受信機能、暗号通信(HTTPSを除く)、リアルタイムでのネットワーク対戦ゲームなどでは、独自に構成したルールを用いる必要があります。

Javaでは、そういったネットワーク通信にも対応できるように標準でいくつかのライブラリが提供されています。

Javaでのネットワーク低水準アクセス

Javaでのネットワーク低水準アクセス(TCP/IPを利用した通信)を利用する場合は、java.net.*のライブラリセットを用います。

課題にとりかかる前に、ネットワークの話をさせていただきます。

OSI参照モデルとTCP/IPで見る高水準アクセスと低水準アクセス

OSI参照モデルは以下のように、ネットワーク通信を役割ごとに7つの階層(レイヤ)に分けた、言わばネットワーク通信の理想型です。

OSI参照モデルでの通信の理想型

OSI参照モデルの各レイヤは、互いに渡すべきデータ形式が決まっている以外、他のレイヤへの干渉はしないことが理想です。

レイヤの構造にしていることでのメリット

各層が機能・役割ごとに独立しており、順序さえ守れば他の層を気にする必要はないという点があります。 順序が決まっていることでどこまでOKか判断できるため、問題点の切り出しが容易です。

TCP/IP

TCP/IPは、OSI参照モデルの各レイヤに対応した、世界標準のプロトコルをまとめたプロトコル・スタックを指します。 名称上、TCPとIPしかないように思われるかもしれませんが、TCP/IPには数多くのプロトコルが含まれます。

OSI参照モデルと照らし合わせると、以下のようになります。

TCP/IPのプロトコルには、OSI参照モデルの複数のレイヤを跨いだプロトコルが存在します。 これらは通信を行うアプリケーションなどの仕様上、敢えてそれぞれのレイヤに分割する必要性がなかったり、分割が不可能な場合があるためです。こればかりはどうしようもないのですが、OSI参照モデルが理想型と言われ、TCP/IPが現実解と言われる理由の一つとして挙げられます。

TCP/IPのプロトコル・スタックでは、アプリケーションレイヤ、トランスポートレイヤ、インターネットレイヤ、ネットワークアクセスレイヤ(リンクレイヤとも呼ばれる)の4つのレイヤで分割されます。この4つのレイヤに分割されているモデルでは、各レイヤを跨いだプロトコルと言うものは存在しません。

例えば、皆さんが今から作成するプログラムでは、セッションの開始、終了や文字コードなどアプリケーション内で定義する必要があります。 アプリケーションの作成を、OSI参照モデルに従うならば、レイヤ7、レイヤ6、レイヤ5に相当するプログラムをそれぞれ作成し、プログラム内でレイヤ間のデータの受け渡し等の処理が必要になります。

ざっくりソケット通信とは

TCP/IPアプリケーション間でデータのやり取りを行うための出入り口のこと。

OSI参照モデルでのセッション層に当たるレイヤで動作し、アプリケーションからは、IPアドレスとポート番号を指定することで通信の確立までを行います。

ソケット通信は、TCP/IPアプリケーションの中核とよんでも差し支えないほど重要な機能で、アプリケーションのプロトコル設計は、ソケット通信の上に作成するようなイメージとなります。

Javaにおけるソケット通信には、java.net.ServerSocketでサーバー側での待受を行い、java.net.Socketがクライアント側とサーバー側を繋ぐ役割を担います。

標準APIのドキュメントは以下のリンク先を参照してください。

  • ServerSocket (Java Platform SE 8 )
  • Socket (Java Platform SE 8 )

クライアント側のプログラム

単純なオウム返しを行うサーバープログラムの作成

新規にプロジェクトを作成し、以下のコードを貼り付けます。

import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.OutputStreamWriter;
import java.io.PrintWriter;
import java.net.ServerSocket;
import java.net.Socket;

public class EchoServer {

  public static void main(String[] args) throws Exception {
    try (
            ServerSocket ss = new ServerSocket(12345);  // サーバー立ち上げ port:12345
            Socket socket = ss.accept(); // クライアントからの接続待ち受け
    ) {
      try (
              BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
              PrintWriter out = new PrintWriter(new OutputStreamWriter(socket.getOutputStream()), true);
              ) {
        // 通信処理自体を無限ループ化
        while (true) {
          // クライアントからのデータ受信 待ち受け
          String message = in.readLine();
          // 「exit」メッセージを受け取ったら無限ループから抜ける
          if (message.equals("exit")) {
            break;
          }
          // メッセージをそのままクライアントに送信
          out.println(message);
        }
      }
    }
  }
}

サーバープログラム側を実行すると、自分のPCのTCPポート12345にサーバーアプリケーションが待受状態になります。(実行後の出力内容はありません。)

TCPMessageClientプロジェクトのメインメソッド内を確認し、接続ポートが12345,接続先ホストがlocalhostになっていることを確認して実行してみましょう。

マルチスレッドで待受する人工無脳サーバーの作成

SuperAIプロジェクトを作成します。

Processor.javaクラスの作成

内容は以下のコードを使ってください。

import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.OutputStreamWriter;
import java.io.PrintWriter;
import java.net.Socket;

class Processor extends Thread {

  java.net.Socket socket;

  public Processor(Socket s) {
    socket = s;
  }

  @Override
  public void run() {
    try (
            BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
            PrintWriter out = new PrintWriter(new OutputStreamWriter(socket.getOutputStream()), true);) {

      out.println("ようこそ！");

      // 通信処理自体を無限ループ化
      while (true) {

        // クライアントからのデータ受信 待ち受け
        String message = in.readLine();

        // 「exit」メッセージを受け取ったら無限ループから抜ける
        if (message.equals("exit")) {
          break;
        }

        if (message.startsWith("こんにちは")) {
          out.println("こんにちは！今日はあいにくの天気ですね……。");
        } else if (message.startsWith("こんばんは")) {
          out.println("今はお昼ですよ？？こんにちは！");
        } else if (message.endsWith("?") || message.endsWith("？")) {
          out.println("その質問にはお答えしかねます。");
        } else if (message.contains("天気")) {
          out.println("今日はいい天気ですよ！");
        } else {
          out.println("おっしゃっている言葉が理解できません……");
        }
      }

      // ソケットを閉じる
      socket.close();

    } catch (Exception e) {
      throw new RuntimeException(e);
    }
  }
}

SuperAI.javaのメインメソッドの作成

import java.net.ServerSocket;
import java.net.Socket;

public class SuperAI {

  public static void main(String[] args) throws Exception {

    try (ServerSocket ss = new ServerSocket(12345)) {
      while (true) {
        Socket s = ss.accept();
        System.out.printf("クライアントからの接続: %s:%d\n", s.getInetAddress(), s.getPort());
        Processor app = new Processor(s);
        app.start();
      }
    }
  }
}

$ ifconfig

TCPクライアントとしてのtelnet,ncコマンド

テキストデータをネットワーク経由で送受信するプログラム

昔より、ネットワーク上でのリモートアクセスなどで使われるプログラムに、telnetと呼ばれるものがあります。

MacOSには、残念ながらHigh Sierra以降削除されています。Sierra以前でも、文字コード関連の重大なバグが含まれており完璧な動作は保証されません。

今回の講義では、TCPMessageClientのプログラムにてその機能を代替させていますが、MacOSには、telnetに代替するncコマンドが用意されています。

telnetの代わりにnc(netcat)を使うには

$ telnet localhost 12345

上記のコマンドは、以下のコマンドに置き換えられます。

$ nc localhost 12345

上記で接続できない場合、-tオプションで、nc(netcat)コマンドをtelnet互換で動作させることができます。

ncコマンドのマニュアルは、$ man ncで確認できます。
