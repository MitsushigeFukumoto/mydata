ソケット通信でテキストデータの送受信ができるようになると、HTTPサーバーが作成できます。

ソケット通信を利用したHTTPサーバーを作成しよう

プロジェクト設定の指定

プロジェクト名: SimpleHTTPServer メインクラス名: jp.ac.ait.k00000.SimpleHTTPServer (k00000は自分の学籍番号に変更する)

クラスやパッケージ、その他ファイルの追加は、必要に応じて任意に追加すること。

簡単な説明

ソケット通信を利用し、HTTPのリクエストデータを受け取りHTTPのレスポンスデータを返す簡易HTTPサーバーを作成します。

HTTP(HyperText Transfar Protocol)は、データの送受信にテキストデータを使います。そのため、今回の講義で行った人工無脳サーバーと同じような仕組みでテキストデータを受け取り、そのデータを解析してHTML,CSS,JavaScriptをHTTPレスポンスの形式で返す仕組みを作成すれば簡単なHTTPサーバーが出来上がります。

せっかくHTTPサーバーを作成するので、クライアントはTCPMessageClientではなく、通常のWebブラウザをクライアントとして使用しましょう。

自分のPCで、ポート番号を12345にしたHTTPサーバープログラムにアクセスするのは、ブラウザでURLをhttp://localhost:12345/とすると(正常に動いていれば)Webページが表示されます。

詳細仕様

サーバーの待受ポート番号は、8088番とします。

HTTPのリクエストデータを受け取ったら、その1行目を判断します。

通常のWebブラウザから送られるHTTPリクエストデータの1行目は以下のようになっています。

GET / HTTP/1.1

これをスペースで区切った3つの文字列を解析します。

上のデータでは、GETはHTTPのリクエストメソッド、/はリクエストパス、 HTTP/1.1は通信に使用するHTTPのバージョンを表します。

今回のプログラムでは、2つめのリクエストパスを確認し、返信するデータを切り替えてもらいます。

データ切り替え仕様

1. リクエストパスの末尾が、.css,.jsじゃない場合、以下のHTMLデータを返すようにしましょう。

3行目の空行が重要なので、忘れず入れるようにしましょう。

HTTP/1.0 200 OK
Content-Type: text/html

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <meta charset="UTF-8">
  <title>SimpleHTTPServer</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <main>
    <h1>このページはSimpleHTTPServerより生成されて返されています。</h1>
    <p><button class="fire">Push!!</button></p>
    <p class="copyright">K00000 - 名前を入力</p>
  </main>
  <script src="script.js"></script>
</body>
</html>

※ 学籍番号や、名前は自分のものに変えてください。

2. リクエストパスの末尾が、.cssだった場合、以下のCSSデータを返すようにしましょう。

3行目の空行が重要なので、忘れず入れるようにしましょう。

HTTP/1.0 200 OK
Content-Type: text/css

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
body {
  height: 100vh;
  display: flex;
  justify-content: center;
  align-items: center;
}
main {
  height: 450px;
  max-height: 90vh;
  width: 800px;
  max-width: 90vw;
  border-radius: 10px;
  box-shadow: rgba(0, 0, 0, 0.1) 0px 20px 60px -10px;
  display: flex;
  justify-content: center;
  align-items: center;
  flex-direction: column;
}
h1 {
  padding: 0 3em;
  margin-bottom: 2em;
  text-align: center;
}
button {
  font-size: 1.25em;
  padding: 0.5em 1em;
}
.copyright {
  margin-top: 20px;
  text-decoration: underline;
  font-style: italic;
}
3. リクエストパスの末尾が、.jsだった場合、以下のJavaScriptデータを返すようにしましょう。

3行目の空行が重要なので、忘れず入れるようにしましょう。

HTTP/1.0 200 OK
Content-Type: text/javascript

var btn = document.querySelector('button.fire');
btn.addEventListener('click', function() {
  alert('Hello, SimpleHTTPServer!!');
});

動作確認

サーバープログラムを実行後、Webブラウザを起動し、http://localhost:8088へアクセスします。

  1. HTMLがきちんと表示されているか
  2. スタイルが設定されているか(CSSが適用されているか)
  3. ボタンをクリックしてアラートが表示されるか(JavaScriptが適用されているか)

上記3点を確認してください。

補足

以下任意で行ってください。

HTMLのデータやCSS、JavaScriptのプログラム等は、自由に改変してもらって良いです。

HTML,CSS,JavaScriptのレスポンスデータは、プロジェクト内にディレクトリを作成し、ファイル読み込みによって返す仕組みにしてもよいです。

課題の提出について

提出するファイルは、実行可能形式のjarファイルと、プロジェクトディレクトリを圧縮したものです。 過去の資料を参考にして、jarファイルの生成を行ってください。
