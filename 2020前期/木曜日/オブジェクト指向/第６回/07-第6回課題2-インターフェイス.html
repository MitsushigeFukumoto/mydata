第6回課題2 - インターフェイス

オブジェクト指向プログラミングおよび演習1 第6回

インターフェイスを用いて、勇者の職業を追加し、その職業に合わせた特殊行動が行えるよう機能追加しましょう

1. IHeroJobインターフェイスの作成

新規クラスの作成より、新規クラス名入力ウィンドウにて、名前:IHeroJob, 種類:Interfaceを選び、「OK」ボタンで新規にインターフェイスを作成します。

IHeroJobのインターフェイスは、以下の抽象メソッドを持ちます。 (この時、public abstract等の修飾子は書きません。)

String getJobName();
String getSpecialAttackName();
AttackResult specialAttack(CharacterBase e);

これらそれぞれのメソッドの詳細は、下記の通りです。

  1. public String getJobName()
    • 職業名を返します。
    • return "自分で考えた職業名をここに"; で良いです。
  2. public String getSpecialAttackName()
    • その職業の特殊行動名を返します。
    • return "自分で考えた特殊行動名をここに"; で良いです。
  3. public AttackResult specialAttack(CharacterBase e)
    • その職業の特殊行動を行います。
    • attack()メソッドを参考に実装を行うと良いでしょう。
    • 今回は、各自で考えた特殊行動を行っていただきますが、「勇者を回復する」、「勇者にダメージを与える」などの処理を入れる場合、SimpleRPG側も大きく変更する必要があるため注意してください。課題を進めることだけに着目するなら、「特定条件を追加して敵に何らかのダメージを与える」処理がもっとも実装しやすいでしょう。

インターフェイスは、あくまでもインターフェイスで、処理の中身の実装はここでは行いません。

2. 職業クラスの追加

新規にHeroクラスを親とした職業クラスを作成し、そのクラスはIHeroJobインターフェイスを実装させます。

新規クラスは、クラス名、メソッド実装内容ともに、各自で自由に考え実装を行います。ただし、Heroクラスを親としているため、コンストラクタ等の実装は必ず行ってください。

職業クラスは、最低2つ自分で追加してください。

[サンプル]剣士クラスの追加

以下のコードは、こちらで用意したサンプル職業です。 これをそのまま使い課題提出することは避けましょう。

クラス名: SwordFighter

package jp.ac.ait.k00000;

import java.util.Random;

public class SwordFighter extends Hero implements IHeroJob {
    public SwordFighter(String name, int hp, int atk, int def, int agi) {
        super(name, hp, atk, def, agi);
    }

    @Override
    public String getJobName() {
        return "剣士";
    }

    @Override
    public String getSpecialAttackName() {
        return "全力攻撃";
    }

    @Override
    public AttackResult specialAttack(CharacterBase e) {
        // 剣士の全力攻撃は50％の確率で成功し、成功時は必ずクリティカルヒット、ダメージ量は3倍となる

        AttackResult ret = new AttackResult();

        Random r = new Random();

        if (r.nextInt(101) > 50) {
            //return 0; // 攻撃失敗
            ret.damage = 0;
            ret.isFailure = true;
            return ret; // 攻撃失敗はクリティカル判定やダメージを与える処理を通さない
        }

        // 攻撃成功の場合はつねにクリティカル
        ret.damage = this.calcAttackPoint(e) * 3;
        ret.isCritical = true;

        // ダメージを与える
        e.damage(ret.damage);

        // 与えたダメージ量を返す
        return ret;
    }

    @Override
    public String toString() {
        // デバッグ用に職業メッセージを追加しておきます。
        return super.toString() + ", " + this.getJobName() + "(" + this.getSpecialAttackName() + ")";
    }
}

詳しい説明は講義内で行います。

3. 勇者の職業決定フローの追加

勇者の生成を行うタイミングで、職業クラスのオブジェクトをHeroオブジェクトとして扱うようにしましょう。

職業決定の方法は各自、「作成した職業からランダムに決定」もしくは、「ユーザーにどの職業が良いかを入力させる」のどちらかを選んでください。

3-1. 作成した職業をランダムに決定する場合

CharacterFactoryクラスのpublic static Hero createHero(String name)メソッドを変更します。

仮に、SwordFighterクラス、SwordFighter2クラス、SwordFighter3クラスの3つの職業が追加されている場合、以下のように変更すればランダムに職業が決定します。

// 勇者を召喚して返す
public static Hero createHero(String name) {

    Random random = new Random();

    // パラメータをランダムに設定
    int hp = random.nextInt(41) + 80;  // 80〜120
    int atk = random.nextInt(8) + 8;  // 8〜15
    int def = random.nextInt(8) + 8;  // 8〜15
    int agi = random.nextInt(8) + 8;  // 8〜15

    // 勇者を生成して返す
    int job = random.nextInt(3);    //3つの職業から選択
    switch (job) {
        case 0: // 剣士
            return new SwordFighter(name, hp, atk, def, agi);
        case 1: // 剣士2
            return new SwordFighter2(name, hp, atk, def, agi);
        default: // それ以外は剣士3
            return new SwordFighter3(name, hp, atk, def, agi);
    }
}

3-2. ユーザーにどの職業が良いかを入力させる場合

ユーザーに「職業を選んでください(1: 剣士, 2: 剣士2, 3:剣士3)」等のメッセージを表示し、入力を受け付け、入力内容を元に作成する職業を決定させましょう。

CharacterFactoryクラスにpublic static Hero createHero(int jobId, String name)メソッドを追加します。 仮に、SwordFighterクラス、SwordFighter2クラス、SwordFighter3クラスの3つの職業が追加されている場合、このメソッドの内容は以下のようにすると良いです。

// 勇者を召喚して返す
public static Hero createHero(int jobId, String name) {

    Random random = new Random();

    // パラメータをランダムに設定
    int hp = random.nextInt(41) + 80;  // 80〜120
    int atk = random.nextInt(8) + 8;  // 8〜15
    int def = random.nextInt(8) + 8;  // 8〜15
    int agi = random.nextInt(8) + 8;  // 8〜15

    // 勇者を生成して返す
    switch (jobId) {
        case 1: // 剣士
            return new SwordFighter(name, hp, atk, def, agi);
        case 2: // 剣士2
            return new SwordFighter2(name, hp, atk, def, agi);
        default: // それ以外は剣士3
            return new SwordFighter3(name, hp, atk, def, agi);
    }
}

呼び出し元は、入力された文字を数値変換して、jobIdとし、引数で渡せばOKです。

4. SimpleRPGの勇者の行動選択処理を変更

現状、勇者の行動選択は、「1で攻撃、それ以外で逃亡」となっています。

これを、「1で攻撃、2でその職業の特殊行動、それ以外で逃亡」とします。

この時、SimpleRPGではHeroクラスのオブジェクトとして勇者を行動させていますが、中身は、各職業の機能が追加された職業クラスのオブジェクトです。

以下のように、職業クラスへのダウンキャストを行うことで、Heroオブジェクトを各職業クラスへと認識させ直したオブジェクトに変換できます。

// 職業の取得 (変数hはHeroのインスタンス)
IHeroJob job = (IHeroJob)h;

// 特殊行動を行う場合は、jobからメソッドを呼び出しましょう。
AttackResult damage_to_e = job.specialAttack(e);

最終的な動作確認を行いましょう

正しい動作が行われているか、きちんと動作確認を行いましょう。
