1. エラー処理と例外処理の練習

以下のプログラムは、ランダムな値(0〜999)を要素に持つ配列を指定された要素数で生成し、その配列から指定されたインデックスの値を取り出して表示するものです。

package jp.ac.ait.k00000.section09;

import java.util.Random;
import java.util.Scanner;

public class Kadai1 {

    /**
     * main
     * @param args
     */
    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

        /*
         * 配列の生成
         */

        // TODO: 【エラー処理】int型の整数値に変換可能な文字列が入力されるまで繰り返したい
        System.out.print("生成する配列の大きさを入力してください > ");
        String input = sc.nextLine();

        // ここは例外処理しない(整数値に変換できる文字列しか来ないはず)
        int size = Integer.parseInt(input);
        int[] arr = createRandomIntArray(size);

        // 生成された配列を画面表示
        System.out.print("生成された配列: ");
        for (int a : arr) {
            System.out.print(a + " ");
        }
        System.out.println();

        /*
         * 配列の要素を取り出し
         */

        // TODO: 【エラー処理】int型の整数値に変換可能な文字列が入力されるまで繰り返したい
        System.out.print("取り出す配列のインデックスを指定してください > ");
        input = sc.nextLine();

        // ここは例外処理しない(整数値に変換できる文字列しか来ないはず)
        int index = Integer.parseInt(input);
        System.out.println("値: " + getValueByIndex(arr, index));
    }
    
    /**
     * 指定された要素数で各要素がランダムな値を持つ配列を生成して返す
     * @param size 要素数
     * @return 生成されたランダムな値を持つ配列オブジェクト
     */
    public static int[] createRandomIntArray(int size) {

        // TODO: 【例外処理】配列の要素数として正しくない値が渡された場合の例外を捕捉し、その場合は要素数0の配列を生成する
        int[] arr = new int[size];

        Random random =  new Random();
        for(int i = 0; i < arr.length; i++) {
            arr[i] =random.nextInt(1000);   // 0〜999まででランダム
        }

        return arr;
    }

    /**
     * 配列とインデックスを渡し、その要素を取り出して返す
     * @param arr 配列のオブジェクト
     * @param index インデックス
     * @return 取り出した要素
     */
    public static int getValueByIndex(int[] arr, int index) {
        // TODO: 【エラー処理】引数arrがnullだった場合、-1を返す
        // TODO: 【例外処理】配列のインデックスとして正しくない値が渡された場合の例外を捕捉し、その場合は-1を返す
        return arr[index];
    }
}

このプログラムにおいて、以下のエラー処理、例外処理を追加してください。

  • public static void main(String[] args)
    • 入力された値がint型の整数値かどうかを判定し、整数値でない場合は再入力させ、正しい値が入力されるまで繰り返す
  • private static int[] createRandomIntArray(int size)
    • 引数sizeが配列の要素数として正しくない値で渡された場合の例外を捕捉し、その場合は要素数0の配列を生成する
  • public static int getValueByIndex(int[] arr, int index)
    • 引数arrがnullの場合、-1を返す
    • 引数indexが引数arrの配列のインデックスとして正しくない値が渡された場合の例外を捕捉し、その場合は-1を返す

　

※メソッドの追加はprivate修飾子をつけて行ってください。ただし、クラスの追加は必要ないかと思われますので、今回は無しとしておきましょう。

プログラムの内容の書き換えも自由に行って構いませんが、メソッドprivate static int[] createRandomIntArray(int size), public static int getValueByIndex(int[] arr, int index)の名前、引数、戻り値の型を変更することは許されません。

また、例外処理の注意点として、Exceptionでのアップキャストを用いた例外処理ではなく、それぞれ詳細な例外を捕捉するようにしてください。

課題1のヒント

  • 文字列が整数値のみで構成されているかのチェック方法には以下の手法があります(他にもあります)
    1. 文字列をcharの配列で取得、1文字ずつCharacter.isDigit(charのインスタンス)でチェックする(マイナスの値をどうするかをよく考えること)
    2. 正規表現(java.util.regexパッケージ以下)を用いる
    3. あえて変換で例外を発生させ、例外が発生した場合は正しくない
  • 入力した整数値文字列がint型の最大値を超えないかを判断する方法
    1. ↑の3番の手法
    2. BigIntegerクラスを使い、compareToでInteger.MIN_VALUE,Integer.MAX_VALUEと比較する
    3. (配列の要素数の最大値は、Integer.MAX_VALUE未満は確実でVMのメモリサイズに依存する)
  • 配列生成時、要素数が最大値を超えないかどうかのチェック方法
    • 厳密には無理(VMのメモリサイズによって異なる場合がある)
    • int[] arr = new int[Integer.MAX_VALUE]; これは確実にjava.lang.OutOfMemoryErrorが発生する
    • 結論、チェックしなくてよい
    • 負の要素数を指定するとjava.lang.NegativeArraySizeExceptionが投げられるのでこれは捕捉すること

　

　

　

2. 実行時例外を拡張して、パスワードの入力チェックにおける例外的状況に対応しましょう。

動作確認のためのメインクラス名は、「Kadai2」として作成します。
パッケージは、「jp.ac.ait.k00000.section09」としてください。

ストーリー

あなたはとある会社の顧客情報を管理するコンピュータ・システム(以下、顧客管理システム)のセキュリティ担当者になりました。

この度、顧客管理システムをWebシステムとして載せ替える開発が進んでおり、現在はユーザー認証として、ユーザーIDとパスワードでの基本的な認証を実装する段階に来ています。

ただ、あなたには一つ心配事があります。この会社の顧客管理システムにアクセスできる権利を持つ社員は、面倒くさがりな社員が非常に多く、システムにログインするパスワードをとても簡単なパスワードにしてしまいます。これでは、外部からどうぞ顧客情報を持って行ってくださいと言っているようなものです。

そのため、パスワードを設定する際に以下のルールを設けることにしました。

  1. パスワードは16文字以上128文字以下とする
  2. パスワードにはアルファベット(a-z, A-Z)と数字(0-9)と一部の記号(",(カンマ)",".(ピリオド)","+(プラス)","-(ハイフン)","*(アスタリスク)","/(スラッシュ)","=(イコール)","_(アンダーバー)","#(井桁)","!(エクスクラメーション)","?(クエスチョン)","@(アットマーク)", " (半角スペース)")が使用できる
  3. パスワードには必ず、大文字のアルファベット、小文字のアルファベット、数字、記号のうちそれぞれ一文字以上使用されること
  4. 容易に推測される文字列を含む事はできない（大文字小文字関係なく、以下の文字列は使用できない）
    1. password
    2. p@ssw0rd
    3. p@55w0rd
    4. qwerty
    5. asdfgh
    6. zxcvbn
    7. abcdef
    8. 123456

これらのルールをすべてクリアする場合はboolean値で「true」、どれか一つでも違反がある場合は自作の例外を投げるプログラムを作成してください。

また、上記の4で指定されている使用できない文字列は、後から拡張できることも考えて、HashSet<String>で持つようにしてください。

自作の例外について

例外のクラス名は、「PasswordRulesViolationException」として作成します。

親クラスとして、検査例外であるjava.lang.Exceptionを継承します。

以下のソースコードを使用してください。変更不要です。

package jp.ac.ait.k00000.section09;

public class PasswordRulesViolationException extends Exception {
  public PasswordRulesViolationException() {
    super();
  }
  public PasswordRulesViolationException(String message) {
    super(message);
  }
  public PasswordRulesViolationException(Throwable cause) {
    super(cause);
  }
  public PasswordRulesViolationException(String message, Throwable cause) {
    super(message, cause);
  }
}

パスワード管理クラスの作成

パスワードがルールどおりに入力されているかをチェックするクラスを作成します。

パッケージ、クラス名

  • jp.ac.ait.k00000.section09.PasswordManager.java

メソッド

  • public static boolean validate(String password) throws PasswordRulesViolationException
    • パスワードが規則を満たしているかどうかをチェックし、満たしている場合はtrueを、満たしていない場合は、PasswordRulesViolationExceptionの例外を投げます。
    • この場合、public PasswordRulesViolationException(String message)のコンストラクタを用いて、簡単な例外メッセージを用意しましょう。

メインメソッド(Kadai2.java)での挙動について

パスワードを標準入力より入力させるようにしてください。

その入力させたパスワードをPasswordManagerクラスのvalidateメソッドにてチェックを行い、PasswordRulesViolationException例外が発生した場合は例外処理内にて、ExceptionのgetMessage()メソッドを活用し、「パスワードのルール基準を満たしていません。getMessage()で取得した詳細説明をここに」と表示して終了してください。

使用可能な記号のリスト

//入力可能な記号をリストで保持
List<Character> symbolList = new ArrayList<>();
symbolList.add(',');
symbolList.add('.');
symbolList.add('+');
symbolList.add('-');
symbolList.add('*');
symbolList.add('/');
symbolList.add('=');
symbolList.add('_');
symbolList.add('#');
symbolList.add('!');
symbolList.add('?');
symbolList.add('@');
symbolList.add(' ');

このリストにある文字は使用してよい記号です。
パスワードの文字を１文字ずつ走査していく場合にリストに存在するかどうかで使用可能な記号かどうかを判定する事ができます。

また、以下のCharacterクラスのメソッドを使うことで、その他のチェックも可能です。

  • 大文字アルファベットかどうかをチェックする場合、Character.isUpperCase(charのインスタンス)メソッド
  • 小文字アルファベットかどうかをチェックする場合、Character.isLowerCase(charのインスタンス)メソッド
  • 数字かどうかをチェックする場合、Character.isDigit(charのインスタンス)メソッド

使っちゃいけない安全でない文字列

Set<String> unusableStringSet = new HashSet<>();
unusableStringSet.add("password");
unusableStringSet.add("p@ssw0rd");
unusableStringSet.add("p@55w0rd");
unusableStringSet.add("qwerty");
unusableStringSet.add("asdfgh");
unusableStringSet.add("zxcvbn");
unusableStringSet.add("abcdef");
unusableStringSet.add("123456");

こちらのSetは使っちゃいけないパスワードです。
なので、1つのパスワードに対して、これらすべてをチェックする必要があります。

Stringのインスタンスには、.containsメソッドがあります。文字列の中に、指定された引数の文字列が含まれている場合Trueを返してくれます。
password.contains( Setから取り出した文字列1つづつ )のように使うと良いでしょう。

また、大文字小文字関係なく、という点にご注意ください。
