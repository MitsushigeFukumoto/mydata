ガイダンス資料

オブジェクト指向プログラミングおよび演習1 ガイダンス資料

はじめに

この講義では、座学と演習を通じてオブジェクト指向プログラミングの基礎を学びます。 講義時間外の成果が露骨に成績に現れる講義です ので、みなさんがんばってください。

当講義の「オブジェクト指向プログラミングおよび演習1」ではオブジェクト指向プログラミング言語である「Java」を用いたプログラミングを学ぶことでオブジェクト指向プログラミングの基本的な考え方についての理解を深めます。
後期に開講する「オブジェクト指向プログラミングおよび演習2」では、オブジェクト指向の考え方を用いて実践的なプログラミングを学ぶ構成となっています。(言語としてJavaを使うとは限りません。)

前期だけ、後期だけという履修でも問題はありませんが、できる限り前後期通して受講するとより理解が深まります。

講義計画

今年度は特例により全13回の講義計画となっております。

講義内容
第1回 ガイダンス（学習の進め方）、Java開発環境の構築、標準入出力を用いた課題
第2回 Java言語の基礎（プリミティブ型、クラス型、文字列型、変数、式、制御構文、配列等）
第3回 オブジェクトとクラス、フィールド、メソッドなど
第4回 クラスの機能１：アクセス制限、カプセル化、多態性(オーバーロード)など
第5回 クラスの機能２：クラスの初期化(コンストラクタ)、クラス型オブジェクト、クラスライブラリなど
第6回 継承、メソッドのオーバーライド、抽象クラス、インターフェイス
第7回 列挙型とコレクションフレームワーク
第8回 クラス内クラスと匿名クラス、ラムダ、StreamAPI、コレクションのソート
第9回 エラー処理と例外処理
第10回 テキストファイル処理とファイルストリーム
第11回 ファイル処理とバイナリデータの取り扱い
第12回 マルチスレッドとネットワークアプリケーション
第13回 Swingを用いたGUIアプリケーション

成績について

成績は各回ごとに出題される課題及びテストにより、以下のように点数を付け、合計 100 点で評価します。

  • 提出物: 70 点
  • テスト: 30 点

不合格の条件

点数の合計が、60 点未満のものは不可(F)です。 最終課題が未提出、または、通常課題が5つ以上未提出の場合は、出席不十分の扱いとなり失(Q)です。

この講義での注意点

  1. 講義自体は、Zoomを用いた遠隔講義にて行います。
    • 重要な連絡はL-Cam,moodle,Teamsを用いて横断的に連絡を行います。どれか一つは必ず反応できるようにしておきましょう。
    • 遠隔講義の注意点は、原則他の講義での注意点と同様ですが、この講義では演習も含むため、画面の切り替え等忙しくなります。画面の切り替えは[⌘] + [tab]で行えます。使いこなせるようにしておきましょう。
  2. この講義では、資料の配布、提出物の管理、テスト等にmoodleを使用します
    • 遠隔講義の増加に伴い、moodle自体が不安定になりアクセス不可能な状態になる場合があります。
    • その場合の代替案としてTeamsを使用しますので、moodleにアクセスできない場合はTeamsを参照するようにしてください。
  3. 資料などは基本的に Mac OS を用いることを基準に作成されています
    • Windows でも問題なく講義に参加できますが、インストールやその他環境に起因するバグ等へのサポートはできかねますのでご注意ください。
  4. 講義内で指定する提出物は必ず出すこと
    • 提出期限は講義を行う日から原則10日後とし、それ以降も提出は可能ですがその場合は提出のみの評価となります。
  5. テキストは指定しません（オススメの入門書は提示いたしますが、購入は強制でありません）
    • 基本構文等は詳しく講義内で行わない場合があります。
    • プログラミングに不慣れな方はおすすめの入門書の何れかを必ず購入し、手元に置いていつでも読めるようにしておきましょう。
  6. 分からないことは必ず聞くこと
    • 講師に聞く以外にも相談できる相手を見つけましょう。また、基本的には講義時間内に聞いてください。
    • 今年度はTeamsを使用するため、そちらの質問チャンネルに質問を投稿しておくのも良いでしょう。(※対応できるのは講義時間内に限ります。)
  7. 講義の進行が速い場合はその場で必ず申し出ること
    • 今回は遠隔講義ということで、録画を残すなどの対応をこちらで行います。
    • あとから復習できるようにはするつもりですが、講義時間中にできる限りフィードバックできることが望ましいため講義時間を有意義に使用しましょう。
  8. メモを取れるようにしておくこと
    • 配布資料にないことを口頭でしゃべる場合が多々あります。いつでもメモを取れるようにしておきましょう。
  9. テストが出題されたら期限内に必ず受験すること
    • 講義後にテストを用意する回があります。課題提出と併せてテストも必ず受験してください。
    • 受験忘れについて毎年相談がありますが、受験期間を過ぎてしまった試験についていかなる理由があろうと個別に再オープンすることはありません。
  10. 講義中、他者に迷惑をかける行為は行わないこと
    • 携帯電話等の使用に関しては、講義を阻害することのないようにお願いします。
    • 生徒同士の相談等はどんどん行ってください。もちろん、TAや講師に質問してもらったほうが良い場合もあります。
    • この講義では出席を取りません。
  11. 当講義にて配布される資料(ソースコード含む)及び動画等のコンテンツについては、講義内、自主学習の範疇での使用のみ許可します。第三者への再配布及び加工を伴う配布を行う場合には担当講師へ問い合わせの上許可を取ること

使用するソフトウェアについて

この講義では次のソフトウェアを使用します。

  1. JavaSDK(JDK)
    • プログラミング言語 Java の開発キット
    • いくつかあるJDKの中から、AdoptOpenJDK ver. 14.0.1-hs を使用します。
    • ※講義進行中にアップデートなど置き換えを行っていただく場合もあります。
  2. IntelliJ IDEA Community Edition
    • Java プログラムの総合開発環境
      • 第2回以降、こちらの開発環境を使用します。
  3. その他、必要に応じて随時アナウンスします。
